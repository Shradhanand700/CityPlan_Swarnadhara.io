A. Transaction Stream Monitor & Fraud

Monitors real-time banking transactions to detect anomalies, duplicate patterns, and suspicious connectivity between accounts.

How the Code Works

Incoming transactions are checked using a Bloom filter for fast duplicate rejection. Sliding-window z-score flags abnormal amounts. LSH-style hashing groups near-duplicate transactions, while BFS explores suspicious account links.

B. Credit Scoring Engine

Combines multiple financial features to compute credit scores using optimization and learning techniques.

How the Code Works

A DP knapsack model selects the best feature combination under constraints. Greedy ranking gives fast approximations. A logistic-regression stub demonstrates learning-based scoring. Feature metadata is retrieved via Trie and HashMap.

// ================= SLIDING WINDOW Z-SCORE =================
struct SlidingZ{
    int W; deque<double> q; double sum=0, sumsq=0;
    SlidingZ(int w):W(w){}
    pair<bool,double> push(double x,double k){
        q.push_back(x); sum+=x; sumsq+=x*x;
        if(q.size()>W){
            double o=q.front(); q.pop_front();
            sum-=o; sumsq-=o*o;
        }
        if(q.size()<10) return {false,0};
        double mean=sum/q.size();
        double var=sumsq/q.size()-mean*mean;
        double sd=sqrt(max(0.0,var));
        double z = sd>0 ? (x-mean)/sd : 0;
        return {z>k,z};
    }
};

// ================= BLOOM FILTER =================
struct Bloom{
    vector<uint8_t> bits; size_t m; int k;
    Bloom(size_t mb,int kh):m(mb),k(kh),bits((mb+7)/8,0){}
};
